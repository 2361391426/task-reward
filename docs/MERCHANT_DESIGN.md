# 商家端设计方案

## 技术选型

### 前端
- **框架**: Vue 3 + TypeScript
- **UI组件库**: Element Plus
- **路由**: Vue Router 4
- **状态管理**: Pinia
- **HTTP客户端**: Axios
- **图表**: ECharts
- **构建工具**: Vite

### 后端
- **语言**: Node.js (推荐) 或 Java Spring Boot
- **框架**: Express / Koa / NestJS
- **数据库**: MySQL 8.0
- **缓存**: Redis
- **文件存储**: 阿里云OSS / 腾讯云COS
- **认证**: JWT

## 核心功能流程

### 1. 创建任务流程
```
填写任务信息 → 预览确认 → 扣除预算（奖励金额 × 总名额）→ 创建成功
```

### 2. 审核流程
```
查看待审核列表 → 点击进入审核详情 → 查看8张截图 →
判断是否符合要求 → 通过/拒绝 →
通过：用户获得奖励，任务剩余名额-1
拒绝：填写拒绝原因，名额不减少
```

### 3. 充值流程
```
选择充值金额 → 选择支付方式 → 跳转支付 →
支付成功 → 回调通知 → 余额增加 → 充值记录
```

### 权限点
- 创建任务

- 删除任务

- 查看财务

## 技术实现要点

### 1. 图片预览优化
- 使用图片懒加载
- 大图预览使用 lightbox 组件
- 支持缩放、旋转
- 预加载下一张图片

### 2. 审核效率优化
- 键盘快捷键支持
  - ← → 切换图片
  - P 通过
  - R 拒绝
  - N 下一条
- 批量审核功能
- 审核进度保存

### 3. 实时数据更新
- WebSocket 推送新提交通知
- 定时刷新待审核数量
- 余额变动实时更新

### 4. 性能优化
- 列表虚拟滚动（大数据量）
- 图片CDN加速
- 接口请求防抖节流
- 路由懒加载

## 开发规范

### 代码规范
- 使用 ESLint + Prettier
- 组件命名使用 PascalCase
- 文件命名使用 kebab-case
- 使用 TypeScript 类型定义

### Git 提交规范
```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
test: 测试
chore: 构建/工具变动
```

### 接口调用规范
- 统一使用 async/await
- 错误统一处理
- Loading 状态管理
- 接口响应缓存（适当场景）

## 部署方案

### 前端部署
- 构建: `npm run build`
- 部署到 Nginx
- 配置 HTTPS
- 配置 CDN 加速

### 后端部署
- 使用 PM2 管理进程
- Nginx 反向代理
- 配置 HTTPS
- 数据库主从复制
- Redis 集群

### 监控告警
- 服务器监控（CPU、内存、磁盘）
- 应用监控（接口响应时间、错误率）
- 业务监控（任务数、提交数、审核数）
- 告警通知（短信、邮件、钉钉）
